네트워크 토폴로지: 컴퓨터 네트워크 요소(element)들(링크, 노드 등)을 물리적으로 연결한 것, 혹은 연결 방식

    즉, 네트워크에 연결된 컴퓨터, 케이블 및 기타 구성 요소의 배치
    -> element를 갖고 있으니 컬렉션이다 / 컬렉션 예시: 리스트, 셋, 맵

    링크: 선(케이블)
    노드: 링크로 연결된 컴퓨터들 (클라우드에서는 터미널)

네트워크: 노드와 링크의 집합

토폴로지(위상 수학): 공간 속의 점(스칼라) 선(벡터) 면(매트릭스)
    빨대: 구멍 1개
    요청: 리퀘스트, 응답: 리스폰스 -> 이름과 방향만 다르지 같음
    레포지토리와 entity는 항상 동기화 / 서비스와 레포지토리는 메소드 호출
    컨텍스트 안에 있는 컴퍼넌트

프로토콜: 위치와 시간의 집합이지만 시간은 클라이언트가 보낼 때라서 위치가 중요함.

TCP vs UDP
    TCP: 연결을 수락하면 통신 회선이 고정 -> Security 설정
        IP와 같이 사용: IP는 무조건 수락
        디폴트: 잠갔다가 여는 것(정상 요청일 때 여는 것)
    UDP: 영상(수락 과정이 없음)

Stream: 물의 흐름(항상 연결 돼 있는 상태)

Client -request-> Server
